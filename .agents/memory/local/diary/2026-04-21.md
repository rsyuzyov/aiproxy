# 2026-04-21

## Root cause chansrv clipboard break + рабочий фикс

Вчерашний wrapper ловушку сработал. Падение воспроизвёл юзер: перевёл ноут в сон → сегодня включил → реконнект → clipboard сломан + 5 мин чёрного экрана.

**Root cause (зафиксирован wrapper'ом):**
```
[2026-04-20T16:44:59] START wrapper_pid=91027 display=:10
[2026-04-20T19:37:39] EXIT wrapper_pid=91027 rc=0 (normal exit)
```
chansrv делает `exit(0)` при **нештатном** обрыве TCP клиента. Не OOM, не signal, не crash — просто сам выходит. Штатный disconnect (close mstsc) chansrv переживает. Подтверждено: `xrdp.log` в 19:37:39 показывает одну строку `Closed socket 30 (AF_UNIX)` — xrdp закрыл UNIX-socket к chansrv, и chansrv сразу exit.

**Дополнительная проблема:** sesman 0.10.x **не перезапускает** chansrv при реконнекте в persistent-сессию. Тест подтвердил: удалил осиротевшие сокеты, юзер реконнект → sesman вызвал только `reconnectwm.sh` (пустой), ни `Calling exec chansrv`, ни cleanup, ничего. xrdp-wm 10 сек ждёт chansrv → `ERROR: error in trans_connect chan` → чёрный экран, потом рабочий стол без clipboard.

## Upstream ресёрч

- Проблема известна с 2016 ([#468](https://github.com/neutrinolabs/xrdp/issues/468)) и 2018 ([#1259](https://github.com/neutrinolabs/xrdp/issues/1259)). В 0.10.x не исправлена.
- Официальная wiki подтверждает: [Copy-Paste-and-network-drives-don't-work-on-a-reconnect](https://github.com/neutrinolabs/xrdp/wiki/Copy-Paste-and-network-drives-don%27t-work-on-a-reconnect). Предлагают только destructive workaround (kill всех xrdp worker'ов).
- Настоящий фикс — [PR #3567 "Close on Reconnect"](https://github.com/neutrinolabs/xrdp/pull/3567), merged **2025-07-21 в devel**. IPC между sesexec и xrdp (Connection Close Protocol). Появится в **0.11+**. В Debian 13 (0.10.1) и 0.10.x бэкпортов нет.

## Решение: патч `/etc/xrdp/reconnectwm.sh`

Sesman официально дёргает этот скрипт на каждый реконнект (передаёт в env DISPLAY, XRDP_SOCKET_PATH, XRDP_PULSE_*_SOCKET, HOME, USER — проверено debug-версией). По умолчанию скрипт пустой.

Моя логика:
1. Парсим DISPLAY → нормализованный номер (`:10.0` и `:10` → `10`)
2. Ищем живой `xrdp-chansrv.real` по `pgrep -f '^/usr/sbin/xrdp-chansrv(\.real)?$'`, фильтр по нормализованному DISPLAY из `/proc/$pid/environ`
3. Если найден — `nothing to do`, exit
4. Если нет — удаляем осиротевшие сокеты (`xrdp_chansrv_socket_N`, audio in/out, xrdpapi_N), стартуем chansrv через `systemd-run --scope` с полным env, ждём появления socket (макс 3с под flock)

**Результат теста (этап 2):**
- Грохнул chansrv `SIGKILL`
- Юзер реконнект → reconnectwm.sh: `chansrv dead → cleaning → starting → socket appeared after 1x0.5s`
- Юзер второй реконнект → reconnectwm.sh: `found alive chansrv pid=193258 → nothing to do`. В `xrdp.log`: `chansrv connect successful` мгновенно, чёрный экран <1 секунды.

Странность на первом реконнекте: "чёрный экран 5 минут" (жалоба юзера), хотя скрипт отработал за 0.5с. В логах не видно причин (нет retry, нет дополнительных timeout'ов). Возможно LXQt/firefox просыпались после 1.5 дней idle. Не воспроизводится в контролируемых условиях.

## Что ещё сделано

- **`EnableFuseMount=false`** в `[Chansrv]` — на LXC без `/dev/fuse` chansrv безрезультатно пытается смонтировать thinclient_drives и спамит `fuse: device not found`. Проброс дисков через RDP в наших контейнерах не нужен.
- Перенесено в [scripts/setup-xrdp.sh](../../../../scripts/setup-xrdp.sh):
  - Функция `configure_reconnect_script()` — устанавливает production reconnectwm.sh через heredoc
  - Блок `EnableFuseMount=false` в `configure_sesman_ini()`
  - Вызов добавлен в `main()` после `configure_sesman_ini`

## Drive redirection включили (уже после того как юзер сам рестартанул ct)

Юзер добавил `features: fuse=1` в `/etc/pve/lxc/107.conf`, рестартанул контейнер. `/dev/fuse` доступен, `fuse3` установлен. В `sesman.ini` `[Chansrv]` вернул `EnableFuseMount=true`.

После `features: fuse=1` и рестарта ct: в mstsc Local Resources → More → Drives → коннект → диски монтируются в `/root/thinclient_drives/root/<DRIVE>`. Проверено — работает.

В [scripts/setup-xrdp.sh](../../../../scripts/setup-xrdp.sh):
- Удалён блок `EnableFuseMount=false` — теперь дефолт xrdp (=true), fuse feature на всех новых контейнерах доступен из коробки если Proxmox разрешил fuse.
- В финальный вывод `main()` добавлен инфо-блок с инструкцией по `features: fuse=1` в LXC-конфиге.
